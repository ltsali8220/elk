1. Architecture Overview

All components run on the same Linux desktop machine. No VM or Docker required. Filebeat collects logs from the OS and ships them to Logstash. Logstash parses and routes them to Elasticsearch. Kibana reads from Elasticsearch to display dashboards.

┌──────────────────────────── Linux Desktop ─────────────────────────────┐ │ │ │ Log Sources Collector Processor Storage/UI │ │ ─────────── ───────── ───────── ────────── │ │ /var/log/syslog ─┐ │ │ /var/log/auth.log─┤ │ │ /var/log/kern.log─┼──► Filebeat ──► Logstash:5044 ──► Elasticsearch │ │ journald ─┤ :5066 :5514 (syslog) :9200 │ │ Docker containers─┘ :5000 (tcp/json) │ │ App log files ───────────────────────────────────────────────────► │ │ │ │ Kibana :5601 │ │ http://localhost:5601 │ └─────────────────────────────────────────────────────────────────────────┘

Log Index Strategy

Index PatternContentsSource
syslog-*System logs, kernel messagesFilebeat → /var/log/syslog, kern.log
auth-logs-*SSH logins, sudo, su, PAMFilebeat → /var/log/auth.log
docker-logs-*Docker container stdout/stderrFilebeat → Docker socket
app-logs-*Application log filesFilebeat → /var/log/app/
journal-logs-*systemd journalFilebeat journald input
misc-logs-*Everything elseFallthrough

2. Prerequisites & System Preparation

2.1 Check system resources

# RAM — need at least 8 GB
free -h

# Disk — need at least 40 GB free
df -h /

# CPU cores (ELK works best with 4+)
nproc

2.2 Update the system

sudo apt-get update && sudo apt-get upgrade -y

2.3 Set vm.max_map_count (required by Elasticsearch)

Elasticsearch needs a higher virtual memory limit than the Linux default.

# Apply immediately
sudo sysctl -w vm.max_map_count=262144

# Make permanent (survives reboot)
echo "vm.max_map_count=262144" | sudo tee -a /etc/sysctl.conf

2.4 Add the Elastic APT repository

One repo covers all ELK components — run this once.

# Import signing key
curl -fsSL https://artifacts.elastic.co/GPG-KEY-elasticsearch \
  | sudo gpg --dearmor -o /usr/share/keyrings/elastic.gpg

# Add repository
echo "deb [signed-by=/usr/share/keyrings/elastic.gpg] \
  https://artifacts.elastic.co/packages/8.x/apt stable main" \
  | sudo tee /etc/apt/sources.list.d/elastic-8.x.list

# Refresh package list
sudo apt-get update
Note — Internet access required These commands download packages from Elastic's servers. Make sure your machine has internet access. Total download is approximately 1.2 GB.

3. Install Elasticsearch

1
Install the package
sudo apt-get install -y elasticsearch=8.13.4

# Lock the version so apt update doesn't upgrade it
sudo apt-mark hold elasticsearch
2
Configure Elasticsearch
sudo nano /etc/elasticsearch/elasticsearch.yml

Replace the entire file with the following (or add these lines):

cluster.name: elk-desktop-lab
node.name: desktop-node-1
network.host: 0.0.0.0
http.port: 9200
discovery.type: single-node

# Security — enabled but no TLS for local lab
xpack.security.enabled: true
xpack.security.http.ssl.enabled: false
xpack.security.transport.ssl.enabled: false
3
Set JVM heap size

Elasticsearch uses 50% of RAM by default. Limit it explicitly to leave room for Kibana and Logstash.

sudo nano /etc/elasticsearch/jvm.options.d/heap.options
# Use half your available RAM, max 8g
# Example for 16 GB machine:
-Xms4g
-Xmx4g

# Example for 8 GB machine:
# -Xms2g
# -Xmx2g
4
Start Elasticsearch and set the elastic password
# Enable and start
sudo systemctl daemon-reload
sudo systemctl enable elasticsearch
sudo systemctl start elasticsearch

# Wait ~30 seconds then check status
sudo systemctl status elasticsearch --no-pager

# Set the elastic superuser password
sudo /usr/share/elasticsearch/bin/elasticsearch-reset-password -u elastic -i
# When prompted, enter:  ElkLab@2024
5
Verify Elasticsearch
curl -u elastic:ElkLab@2024 http://localhost:9200/_cluster/health?pretty

Expected output — "status" : "green" :

{
  "cluster_name" : "elk-desktop-lab",
  "status" : "green",
  "number_of_nodes" : 1,
  "number_of_data_nodes" : 1,
  ...
}
Note — single-node health On a single node you may see "status": "yellow" after adding data. This is normal — it means replica shards cannot be assigned (no second node). All your data is safe.

4. Install Logstash

1
Install the package
sudo apt-get install -y logstash=1:8.13.4-1
sudo apt-mark hold logstash
2
Configure Logstash settings
sudo nano /etc/logstash/logstash.yml
http.host: "0.0.0.0"
xpack.monitoring.enabled: false
config.reload.automatic: true
config.reload.interval: 30s
pipeline.workers: 2
3
Set JVM heap for Logstash
sudo nano /etc/logstash/jvm.options

Find and update the heap lines:

-Xms512m
-Xmx1g
4
Create the Logstash internal user in Elasticsearch

Logstash needs a write-only account to index logs. Run these curl commands:

# Create a role that can write to log indices only
curl -u elastic:ElkLab@2024 -X PUT \
  http://localhost:9200/_security/role/logstash_writer \
  -H "Content-Type: application/json" -d '{
    "cluster": ["manage_index_templates", "monitor"],
    "indices": [{
      "names": ["syslog-*","auth-logs-*","docker-logs-*",
                 "app-logs-*","journal-logs-*","misc-logs-*"],
      "privileges": ["write","create_index","create"]
    }]
  }'

# Create the logstash_internal user with that role
curl -u elastic:ElkLab@2024 -X PUT \
  http://localhost:9200/_security/user/logstash_internal \
  -H "Content-Type: application/json" -d '{
    "password": "LogstashLab@2024",
    "roles":    ["logstash_writer"],
    "full_name": "Logstash Internal"
  }'
5
Enable Logstash
sudo systemctl enable logstash

Do not start it yet — configure the pipeline first (Section 5).

5. Configure Logstash Pipelines

Pipeline files go in /etc/logstash/conf.d/. Logstash reads all .conf files in that directory in alphabetical order.

5.1 Input — 01-input.conf

sudo nano /etc/logstash/conf.d/01-input.conf
input {
  # Filebeat agent sends logs here
  beats {
    port => 5044
  }

  # Any app sending JSON over TCP
  tcp {
    port  => 5000
    codec => json
    tags  => ["tcp_json"]
  }

  # Traditional syslog (UDP + TCP)
  syslog {
    port => 5514
    tags => ["syslog_raw"]
  }
}

5.2 Auth log filter — 10-auth-filter.conf

sudo nano /etc/logstash/conf.d/10-auth-filter.conf
filter {
  if "linux_auth" in [tags] {
    grok {
      match => {
        "message" => [
          "%{SYSLOGTIMESTAMP:syslog_ts} %{SYSLOGHOST:host_name} %{DATA:proc}(?:\[%{POSINT:pid}\])?: %{GREEDYDATA:auth_msg}",
          "%{SYSLOGTIMESTAMP:syslog_ts} %{SYSLOGHOST:host_name} %{GREEDYDATA:auth_msg}"
        ]
      }
    }
    date {
      match  => ["syslog_ts", "MMM  d HH:mm:ss", "MMM dd HH:mm:ss"]
      target => "@timestamp"
    }
    mutate {
      rename => { "proc"     => "[process][name]" }
      rename => { "pid"      => "[process][pid]"  }
      rename => { "auth_msg" => "message"          }
    }
    # Tag specific auth events
    if [message] =~ /Accepted (password|publickey)/ {
      mutate { add_tag => ["ssh_login_success"] }
    }
    if [message] =~ /Failed password/ {
      mutate { add_tag => ["ssh_login_failure"] }
    }
    if [message] =~ /sudo.*COMMAND/ {
      mutate { add_tag => ["sudo_command"] }
    }
    if [message] =~ /session opened/ {
      mutate { add_tag => ["session_open"] }
    }
    if [message] =~ /session closed/ {
      mutate { add_tag => ["session_close"] }
    }
  }
}

5.3 Syslog filter — 20-syslog-filter.conf

sudo nano /etc/logstash/conf.d/20-syslog-filter.conf
filter {
  if "syslog" in [tags] or "syslog_raw" in [tags] {
    grok {
      match => {
        "message" => "%{SYSLOGTIMESTAMP:syslog_ts} %{SYSLOGHOST:host_name} %{DATA:proc}(?:\[%{POSINT:pid}\])?: %{GREEDYDATA:syslog_msg}"
      }
      overwrite => ["message"]
    }
    date {
      match  => ["syslog_ts", "MMM  d HH:mm:ss", "MMM dd HH:mm:ss"]
      target => "@timestamp"
    }
    mutate {
      rename => { "syslog_msg" => "message"        }
      rename => { "proc"       => "[process][name]" }
      rename => { "pid"        => "[process][pid]"  }
    }
    if [process][name] in ["sshd","sudo","su","login","passwd","useradd","usermod"] {
      mutate { add_tag => ["linux_auth"] }
    }
    if [process][name] in ["kernel"] {
      mutate { add_tag => ["kernel_event"] }
    }
  }
}

5.4 Docker log filter — 30-docker-filter.conf

sudo nano /etc/logstash/conf.d/30-docker-filter.conf
filter {
  if "docker_log" in [tags] {
    mutate {
      add_field => { "[event][dataset]" => "docker" }
    }
    if [container][name] {
      mutate {
        rename => { "[container][name]" => "[service][name]" }
      }
    }
    # Try to parse JSON app logs from containers
    if [message] =~ /^\{/ {
      json {
        source => "message"
        target => "parsed"
        skip_on_invalid_json => true
      }
    }
  }
}

7. Install Filebeat

Filebeat is the lightweight log shipper. It reads log files and sends them to Logstash.

1
Install the package
sudo apt-get install -y filebeat=8.13.4
sudo apt-mark hold filebeat
2
Write the Filebeat config
sudo nano /etc/filebeat/filebeat.yml

Replace the entire default config with this:

filebeat.inputs:

  # ── System logs ─────────────────────────────
  - type: log
    enabled: true
    paths:
      - /var/log/syslog
      - /var/log/messages
    tags: ["syslog"]
    fields:
      log_type: syslog
      host_type: linux-desktop
    fields_under_root: false
    multiline.pattern: '^[A-Z][a-z]{2}\s'
    multiline.negate: true
    multiline.match: after

  # ── Authentication logs ─────────────────────
  - type: log
    enabled: true
    paths:
      - /var/log/auth.log
      - /var/log/secure
    tags: ["linux_auth"]
    fields:
      log_type: auth
      host_type: linux-desktop
    fields_under_root: false

  # ── Kernel logs ─────────────────────────────
  - type: log
    enabled: true
    paths:
      - /var/log/kern.log
    tags: ["syslog", "kernel_event"]
    fields:
      log_type: kernel
    fields_under_root: false

  # ── Application logs (add your app paths here)
  - type: log
    enabled: true
    paths:
      - /var/log/nginx/*.log
      - /var/log/apache2/*.log
      - /var/log/app/*.log
    tags: ["app_log"]
    fields:
      log_type: application
    fields_under_root: false
    ignore_older: 24h

  # ── Docker container logs ────────────────────
  - type: container
    enabled: true
    paths:
      - /var/lib/docker/containers/*/*.log
    stream: all
    tags: ["docker_log"]
    fields:
      log_type: docker
    fields_under_root: false
    processors:
      - add_docker_metadata:
          host: "unix:///var/run/docker.sock"

  # ── systemd journal ──────────────────────────
  - type: journald
    enabled: true
    id: journald-local
    tags: ["journal"]
    fields:
      log_type: journal
    fields_under_root: false

# ── Processors ──────────────────────────────────
processors:
  - add_host_metadata:
      when.not.contains.tags: forwarded
  - add_locale: ~

# ── Output: send to Logstash ─────────────────────
output.logstash:
  hosts: ["localhost:5044"]

# ── Kibana (for dashboard setup) ─────────────────
setup.kibana:
  host: "localhost:5601"
  username: "elastic"
  password: "ElkLab@2024"

logging.level: info
logging.to_files: false
3
Fix permissions and add filebeat to Docker group
# Fix config file permissions (filebeat security requirement)
sudo chmod go-w /etc/filebeat/filebeat.yml
sudo chown root:root /etc/filebeat/filebeat.yml

# Allow Filebeat to read Docker logs (if Docker is installed)
sudo usermod -aG docker filebeat

# Allow Filebeat to read system logs
sudo usermod -aG adm filebeat
4
Test config and start Filebeat
# Validate config
sudo filebeat test config -c /etc/filebeat/filebeat.yml

# Test connection to Logstash
sudo filebeat test output -c /etc/filebeat/filebeat.yml

# Enable and start
sudo systemctl enable filebeat
sudo systemctl start filebeat
sudo systemctl status filebeat --no-pager

8.2 Add a custom application log

To monitor a log file from your own application, edit /etc/filebeat/filebeat.yml and add a new input block:

  - type: log
    enabled: true
    paths:
      - /home/youruser/myapp/logs/*.log    # ← change to your app's log path
    tags: ["app_log"]
    fields:
      app_name: myapp
      log_type: application
    fields_under_root: false

Then restart Filebeat: sudo systemctl restart filebeat

8.3 Enable auditd (advanced — system call auditing)

# Install auditd
sudo apt-get install -y auditd audispd-plugins

# Add rules to monitor sensitive files and commands
sudo auditctl -w /etc/passwd   -p wa -k user_modify
sudo auditctl -w /etc/shadow   -p wa -k user_modify
sudo auditctl -w /etc/sudoers  -p wa -k sudo_modify
sudo auditctl -w /bin/su       -p x  -k privilege_escalation

# Make rules permanent
sudo auditctl -l | sudo tee /etc/audit/rules.d/elk-lab.rules
sudo systemctl restart auditd

8.4 Monitor a PostgreSQL log

# Enable PostgreSQL logging
sudo nano /etc/postgresql/*/main/postgresql.conf

Set these values:

log_destination = 'stderr'
logging_collector = on
log_directory = 'log'
log_filename = 'postgresql-%Y-%m-%d.log'
log_line_prefix = '%t [%p]: [%l-1] user=%u,db=%d,app=%a,client=%h '
log_connections = on
log_disconnections = on
log_duration = off
log_statement = 'ddl'

Then add the Postgres log path to Filebeat:

  - type: log
    enabled: true
    paths:
      - /var/lib/postgresql/*/main/log/postgresql-*.log
    tags: ["app_log"]
    fields: { app_name: postgresql, log_type: database }
    fields_under_root: false

9. Create Kibana Index Patterns & Dashboards

9.1 Create index patterns via the API

Run these curl commands once Kibana is running. They create all the data views automatically.

KIBANA="http://localhost:5601"
AUTH="elastic:ElkLab@2024"
H='-H "kbn-xsrf: true" -H "Content-Type: application/json"'

for PATTERN in "auth-logs-*" "syslog-*" "docker-logs-*" "app-logs-*" "journal-logs-*" "misc-logs-*"; do
  curl -s -u "$AUTH" -X POST "$KIBANA/api/data_views/data_view" \
    -H "kbn-xsrf: true" -H "Content-Type: application/json" \
    -d "{\"data_view\":{\"title\":\"$PATTERN\",\"timeFieldName\":\"@timestamp\"}}"
  echo " → Created: $PATTERN"
done

9.2 Create index patterns in Kibana UI (alternative)

  1. Open Kibana → Stack Management → Data Views → Create data view
  2. Enter pattern (e.g. auth-logs-*) and set time field to @timestamp
  3. Repeat for each pattern in the table below
PatternTime fieldIndex in Discover
auth-logs-*@timestampSSH logins, sudo, PAM
syslog-*@timestampSystem + kernel events
docker-logs-*@timestampContainer stdout/stderr
app-logs-*@timestampNginx, PostgreSQL, custom apps
journal-logs-*@timestampsystemd journal
misc-logs-*@timestampEverything else

Pipeline hot-reload

Logstash reloads pipeline files automatically every 30 seconds. After editing any .conf file in /etc/logstash/conf.d/, just wait 30 seconds — no restart needed.

Adding a new log source

  1. Edit /etc/filebeat/filebeat.yml — add a new input block
  2. Run sudo filebeat test config to validate
  3. Run sudo systemctl restart filebeat
  4. Add a matching filter in /etc/logstash/conf.d/ if needed
  5. Add a new output route in 99-output.conf if needed
  6. Create a new data view in Kibana for the new index pattern

12. Troubleshooting

Elasticsearch won't start

# Read the startup error
sudo journalctl -u elasticsearch --since "5 minutes ago" --no-pager

# Most common fix — vm.max_map_count too low
sudo sysctl -w vm.max_map_count=262144
